## 📋 Как работает?

### 1️⃣ **ORM (Object-Relational Mapping)**
Django ORM - это "переводчик" между Python кодом и SQL:
- **Python классы** (models.py) = **Таблицы БД**
- **Атрибуты класса** = **Столбцы таблицы**
- **Экземпляры класса** = **Строки данных**

Пример:
```python
# Это в Python...
class Product(models.Model):
    name = models.CharField(max_length=200)
    price = models.DecimalField()

# ...становится вот этой SQL таблицей:
CREATE TABLE store_product (
    id INTEGER PRIMARY KEY,
    name VARCHAR(200),
    price DECIMAL
)
```

## 🔗 Как таблицы связаны между собой?

### **Один-к-Многим (1:M)**
```
Department (1) ────────→ Product (M)
                 один отдел может иметь много товаров

Category (1) ────────→ Product (M)
              одна категория может иметь много товаров

Color (1) ────────→ ProductImage (M)
          один цвет может быть на многих фото

Color (1) ────────→ ProductVariant (M)
          один цвет может быть в разных вариациях

Size (1) ────────→ ProductVariant (M)
         один размер может быть у разных вариаций

Product (1) ────────→ ProductImage (M)
            один товар может иметь много фото

Product (1) ────────→ ProductVariant (M)
            один товар может иметь много вариаций

Product (1) ────────→ OrderItem (M)
            один товар может быть во многих заказах

User (1) ────────→ Order (M)
         один пользователь может иметь много заказов

Order (1) ────────→ OrderItem (M)
          один заказ может иметь много товаров
```

### **Один-к-Одному (1:1)**
```
User (1) ────────→ Favorite (1)
         один пользователь = одно избранное

User (1) ────────→ EmailVerification (1)
         один пользователь = один код подтверждения
```

### **Многие-ко-Многим (M:M)**
```
Product (M) ←──────→ Favorite (M)
            один товар может быть у многих пользователей
            в избранном
```

## 📊 Пример: Полный жизненный цикл заказа

### 1️⃣ **Пользователь регистрируется:**
```
1. User создается (is_active=False)
2. EmailVerification создается с кодом 482916
3. Пользователь получает email с кодом
4. Вводит код → User.is_active = True
5. Код удаляется из таблицы
```

### 2️⃣ **Пользователь просматривает товары:**
```
Product.objects.filter(available=True)
    ├─ Получаем все товары
    ├─ Для каждого товара показываем:
    │   ├─ Department (отдел)
    │   ├─ Category (категория)
    │   ├─ ProductImage (фото разных цветов)
    │   └─ ProductVariant (остатки размер+цвет)
```

### 3️⃣ **Добавляет товар в избранное:**
```
Favorite.products.add(product_id)
    ├─ Находим избранное пользователя
    └─ Добавляем товар (M2M связь)
```

### 4️⃣ **Оформляет заказ:**
```
1. Создаем Order:
   - user_id, first_name, last_name
   - email, phone, address, city
   - status = 'new'
   - payment_method = 'kaspi'
   - total_price = 45000

2. Для каждого товара в корзине создаем OrderItem:
   - order_id = 1
   - product_id = 3
   - price = 15000 (цена на момент заказа)
   - quantity = 2

3. Уменьшаем ProductVariant.stock
```

### 5️⃣ **Администратор видит в админ-панели:**
```
- Заказ №1
  - Статус: new → paid → shipped → completed
  - Способ оплаты: Kaspi Pay
  - Итого: 45 000₸
  - Товары:
    - Рубашка синяя (2 шт.) × 15 000₸
    - Куртка красная (1 шт.) × 15 000₸
```

## 🛠️ Команды для работы с БД в терминале

### **Просмотр всех данных:**
```bash
# Запустить Django shell
python manage.py shell

# Все категории
from store.models import Category
for cat in Category.objects.all():
    print(cat.name)

# Все заказы
from store.models import Order
for order in Order.objects.all():
    print(f"Заказ #{order.id}: {order.total_price}₸")

# Товары конкретной категории
from store.models import Product, Category
category = Category.objects.get(name="Рубашки")
for product in category.products.all():
    print(f"{product.name} - {product.price}₸")
```

### **Создание миграций (изменения в структуре):**
```bash
# Создать миграцию (сохраняет изменения)
python manage.py makemigrations

# Применить миграцию (обновляет БД)
python manage.py migrate
```
